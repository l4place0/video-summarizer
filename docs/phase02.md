# Phase 02 — Web UI

## 目标
最小可用的 Web 界面，调用 Phase 01 的后端 API，实现视频摘要的可视化操作。

## 技术选型
- 纯静态前端（HTML + CSS + JS），无构建工具
- 通过 FastAPI 的 `StaticFiles` 挂载，与后端同源同端口
- 无框架依赖，原生 JS 实现交互

## 模块清单

### 1. 静态文件服务 (`app/web/`)
- `index.html` — 单页应用
- `style.css` — 样式
- `app.js` — 交互逻辑
- 通过 FastAPI `StaticFiles` 挂载到 `/`

### 2. 页面布局
单页三区：

```
┌─────────────────────────────────────────┐
│  Header: 视频摘要工具                     │
├─────────────────────────────────────────┤
│  输入区:                                  │
│  [ URL 输入框 ] [语言▾] [Provider▾] [提交] │
├─────────────────────────────────────────┤
│  结果区:                                  │
│  状态: summarizing ████░░░ 60%           │
│  标题: Linux网络命名空间核心               │
│  摘要: 这段视频主要讲解了...               │
│  [展开转录原文]                           │
├─────────────────────────────────────────┤
│  历史区:                                  │
│  ┌──────┬──────────┬──────┬──────┐      │
│  │ 时间  │ 标题      │ 状态  │ 操作 │      │
│  ├──────┼──────────┼──────┼──────┤      │
│  │ ...  │ ...      │ done │ 查看 │      │
│  └──────┴──────────┴──────┴──────┘      │
│  [清理存储▾]                              │
└─────────────────────────────────────────┘
```

### 3. 交互逻辑 (`app.js`)

#### 提交任务
- 校验 URL 非空
- POST `/api/summarize`，获取 task_id
- 立即进入轮询

#### 状态轮询
- 每 2 秒 GET `/api/tasks/{task_id}`
- 更新状态指示器（文字 + 进度条动画）
- 状态 done → 展示摘要；failed → 展示错误

#### 任务历史
- 页面加载时 GET `/api/tasks` 填充表格
- 点击"查看"加载该任务详情到结果区
- 按创建时间倒序

#### 存储管理
- 页面加载时 GET `/api/storage` 展示占用
- "清理存储"按钮，带确认弹窗
- 支持全量清理

### 4. 后端改动 (`app/main.py`)
- 挂载静态文件目录
- 根路径 `/` 返回 `index.html`

```python
from fastapi.staticfiles import StaticFiles

app.mount("/", StaticFiles(directory="app/web", html=True), name="web")
```

注意：静态文件挂载需在 API 路由之后，避免覆盖 `/api/*` 和 `/health`。

### 5. 样式要求
- 响应式，移动端可用
- 深色主题（低亮度护眼）
- 状态用颜色区分：pending=灰, downloading/transcribing/summarizing=蓝, done=绿, failed=红

## 验收标准

### 功能验收

| # | 场景 | 预期结果 |
|---|------|---------|
| 1 | 访问 `localhost:8000` | 显示 Web 页面 |
| 2 | 输入 URL 并提交 | 任务创建，状态实时更新 |
| 3 | 任务完成 | 摘要正确展示，含标题和元数据 |
| 4 | 任务失败 | 错误信息展示，红色状态 |
| 5 | 页面刷新 | 历史任务列表加载 |
| 6 | 点击历史任务 | 详情加载到结果区 |
| 7 | 清理存储 | 确认后执行，占用清零 |
| 8 | 无效 URL 提交 | 前端校验拒绝，不发请求 |

### 非功能验收

| 维度 | 要求 |
|------|------|
| 加载 | 页面首屏 < 1s |
| 兼容 | Chrome / Firefox 最新版 |
| 响应式 | 移动端可用，布局自适应 |
| 无依赖 | 不引入任何 JS/CSS 框架 |

```bash
# - 清理存储，确认生效
```
