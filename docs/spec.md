# 开发规范

## 迭代闭环流程

每个 Phase 实现完成后，必须通过以下自检才能提交验收。

### Step 1: 单元测试
- 各模块独立测试，mock 外部依赖
- `python -m pytest tests/ -v` 全部通过，0 warnings

### Step 2: 集成测试（业务流自测）
- Mock 外部依赖，通过真实 HTTP 请求跑完整业务流程
- 覆盖：成功路径、错误路径、状态流转、数据持久化、多 provider

### Step 3: 真实环境验证
- 在当前可用的真实环境下，跑完整 pipeline 验证
- 外部服务配置（API 端点、模型名、参数格式）必须通过查询实际服务来确认，不靠假设
- 环境不具备的部分明确标记为"未验证"，不假设通过

### Step 4: SOP 验证
- 自己跑一遍所有验收命令，确认可执行且输出正确
- SOP 命令必须可直接复制执行，不得有格式或解析问题
- 代码修改后必须重启服务并重新验证，不假设变更生效

### Step 5: 验收交付
- pytest 报告（全绿）
- 真实环境验证结果
- 已知限制清单

## 核心原则
**自测覆盖度必须 >= 用户验收覆盖度。用户不应发现自测应该发现的问题。**

## 代码规范
- Python 3.12+, uv, FastAPI
- 类型注解完整
- 不写多余注释
